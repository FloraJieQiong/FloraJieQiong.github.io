数据结构2-线性结构——2-4-应用实例：多项式加法运算

多项式加法运算

typedef struct PolyNode{
    int coef;//系数
    int expon;//指数
    struct Node *link;//指向下一个节点的指针
}*Polynomial;
Polynomial P1,P2;

算法思路: 两个指针P1，P2分别 指向 多项式 的第一个结点，不断循环：

  • P1->expon==P2->expon:系数相加，若 结果不为0，则作为 结果多项式 对应项 的系数。同时，P1和P2都分别指向下一项；
  • P1->expon > P2->expon:将P1的当前项 存入 结果多项式，并使P1指向下一项；
  • P1->expon < P2->expon:将P2的当前项 存入 结果多项式，并使P2指向下一项。
  • 当 某一个多项式 处理完时，将另一个多项式 的所有结点 依次复制到 结果多项式中去。
Ploynomial PolyAdd(Polynomial P1, Polynomial P2){
    Polynomial front,rear,temp;//front 记录 结果多项式 链表 头结点，rear 结果多项式 尾
    int sum;

    rear = (Polynomial)malloc(sizeof(PolyNode));//为方便表头插入，先产生一个 临时空结点 作为 结果多项式 链表头
    front = rear;

    while(P1 && P2){
        //当两个多项式 都有 非零项 待处理时
        switch(Compare(P1->expon, P2->expon)){
            case 1:
                //P1->expon > P2->expon
                Attach(P1->ceof, P1->expon, &rear);//将 P1的系数和指数 拷贝到结果多项式中，接到 rear的后边
                P1 = P1->link;//P1后移
                break;
            case -1:
                Attach(P2->ceof, P2->expon, &rear);
                P2=P2->link;
                break;
            case 0:
                sum = P1->ceof + P2->ceof;
                if(sum){
                    //sum != 0
                    Attach(sum, P1->expon, &rear);
                }
                P1=P1->link;
                P2=P2->link;
                break;
        }
    }
    //将 未处理完的 另一个多项式 的所有节点 依次复制到 结果多项式 中
    for(; P1; P1=P1->link){
        //P1 不空
        Attach(P1->ceof, P1->ceof, &rear);
    }
    for(; P2; P2=P2->link){
        //P2 不空
        Attach(P2->ceof, P2->expon, &rear);
    }

    rear->link = NULL;//rear 是结果多项式的尾，指向NULL
    temp = front;
    front = front->link;//令front指向 结果多项式 第一个 非零项
    free(temp);//释放临时 空表头结点
    return front;
}


void Attach(int ceof, int expon, Polynomial *PtrRear){
    //ceof 系数，expon 指数，*PtrRear 结果多项式 尾指针
    //需要改变 结果多项式 尾指针的值，所以 第三个参数是 结点指针的地址，*PtrRear指向尾项

    Polynomial P;

    P = (Polynomial)maoolc(sizeof(PolyNode))//申请新结点
    P->ceof = ceof;//对新结点 赋值
    P->expon = expon;
    (*PtrRear)->link = P;//将P指向的 新结点 插入到 结果表达式 尾项 的后边
    *PtrRear = P;//修改PtrRear值
}

数据结构2-线性结构——2-3-队列

2.3.1 什么是队列

什么是队列

队列：具有一定操作约束的线性表
插入、删除操作：只能在一端 插入，另一端 删除。
入队：数据插入
出队：数据删除
顺序：先来先服务、先进先出（FIFO）

队列的顺序存储实现

由一个数组 和 记录 头元素位置 的变量 front 以及 一个 记录 尾元素位置 都变量 rear组成。

#define MaxSize<存储数据元素的最大个数>
typedef struct{
    ElementType Data[MaxSize];
    int rear;
    int front;
}Queue;

没有元素时，front和rear都指向-1
第一个元素下表是0

当没有元素时，front = rear = 0

当前有5个元素，再添加一个元素，front = rear

思考：
（1）这种方案，堆栈空和满 的判别条件是什么？
（2）为什么会出现空、满 无法区分？根本原因是什么？
rear、front距离是[0, n-1].当前例子中，rear front的距离是[0,5]
当堆栈空，满时，距离都为0
根本原因：
rear、front距离[0,n-1]，共n种；
当前队列状态：空，1个元素，2个元素，3,4,5,6.共n+1种状态
用n种距离 表示不了 n+1种状态

解决方案：

  • （1）使用 额外标记：Size或tag
    Size标记元素个数，Size = 0时，队列空，Size为n时，队列满
    tag，当rear=front，分不清是空还是满时，用tag标记最后一次操作，如果 插入，tag为1，删除tag为0.看最后一次操作的tag，判断是空还是满。
  • （2）仅使用n-1个数组空间，不放满。当前例子中，最多只放5个元素。

采取第二种方案：

(1)入队列

void AddQ(Queue *PtrQ, ElementType item){
    if((PtrQ->rear + 1) % MaxSize == PtrQ->front){
        printf("队列满");
        return;
    }
    PtrQ -> rear = (PtrQ->rear + 1) % MaxSize;
    PtrQ -> Data[PtrQ->rear] = item;
}

(2)出队列

ElementType DeleteQ(Queue *PtrQ){
    if(PtrQ -> front == PtrQ->rear){
        printf("队列空");
        return ERROR;
    } else {
        PtrQ->front = (PtrQ->front+1)%MaxSize;
        return PtrQ->Data[PtrQ->front];
    }
}

2.3.2 队列的 链式 存储实现

可以用 单链表 实现。
插入和删除 操作 分别在 链表 的两头进行。
队列指针 front和rear 分别 指向链表的哪一头？
front是用来删除的，rear是用来插入的
链表的头，可以做front或rear，插入 删除都可以；
链表的尾，如果做front，删除了，就找不到链表的头了。
所以 链表的头做front，头做rear

typedef struct Node{
    ElementType Data;
    struct Node *Next;
}QNode;
typedef struct{
//链队列结构
QNode *rear;//指向队尾结点
QNode *front;//指向队头结点
}LinkQueue;
LinkQueue *PtrQ;

不带头结点 的链式队列 出队：

ElementType DeleteQ(LinkQueue *PtrQ){
    QNode *FrontCell;
    ElementType FrontElem;
    if(PtrQ->front == NULL){
        printf("队列空");
        return ERROR;
    }
    FrontCell = PtrQ->front;
    if(PtrQ->front == PtrQ->rear){
        //队列只有一个元素
        PtrQ->front = PtrQ->rear = NULL;//删除后 队列 置空
    } else {
        PtrQ->front = PtrQ->front->Next;
    }
    FrontElem = FrontCell->data;
    free(FrontCell);//释放 被删除 结点 的空间
    return FrontElem;
}

数据结构2-线性结构——2-2-堆栈

2.2.1

什么是堆栈？

算术表达式

  • 组成
    • 运算数：1,2,3
    • 运算符号：+，-，*，/

后缀表达式

  • 中缀表达式：运算符号位于两个运算数之前。a+b*c-d/e
  • 后缀表达式：运算符号位于两个运算数之后。abc*+de/-

计算：62/3-42*+=?
后缀表达式求值策略：从左往右扫描，逐个处理运算数和运算符号。
1.遇到运算数先记住
2.遇到运算符，将与运算符挨着的2个运算数计算
答案是8，T(n)=O(n)

用什么数据结构存储后缀表达式？
需要一种存储方法，能顺序存储运算数，并在需要时倒序输出。
堆栈

堆栈的抽象数据类型描述

堆栈：具有一定操作约束的线性表
只在一端（栈顶，Top）做插入删除

  • 入栈：插入数据
  • 出栈：删除数据
  • 顺序：后入先出（Last In First Out ，LIFO）

堆栈的抽象数据类型描述

2.2.2 栈的顺序存储实现

通常由一个一维数组和一个记录栈顶元素位置的变量组成

#define MaxSize<储存数据元素的最大个数>
typedef struct{
    ElementType Data[MaxSize];
    int Top;
}Stack;

(1)入栈

void Push(Stack *PtrS, ElementType item){
    if(Ptrs -> Top == MaxSize - 1){
        printf("堆栈满");
        return;
    } else {
        PtrS -> Data[++(PtrS -> Top)] = item;
        return;
    }
}

(2)出栈

ElementType Pop(Stack *PtrS){
    if(PtrS -> Top == -1){
        printf("堆栈空");
        return ERROR;//ERROR是ElementType的特殊值，标志错误
    } else {
        return (PtrS -> Data[(PtrS -> Top)--]);
    }
}

[例]请用一个数组实现两个堆栈，要求最大地利用数组空间，使数组只要有空间入栈操作就可以成功。

[分析]两个栈 分别从 数组的两头开始 向中间生长，当两个栈的 栈顶指针相遇时，两个栈都满了。

#define MaxSize<存储数据元素的最大个数>
struct DStack{
    ElementType Data[MaxSize];
    int Top1;//堆栈1的栈顶指针
    int Top2;//堆栈2的栈顶指针    
}S;
//堆栈空
S.Top1 = -1;
S.Top2 = MaxSize;

(1)入栈

void Push(struce DStack *PtrS, ElementType item, int Tag){
    //Tag作为区分两个堆栈的标志，取值为1,和2
    if(PtrS -> Top2 - PtrS -> Top1 == 1){
        printf("堆栈满");
        return;            
    }
    if(Tag == 1){
        //对 堆栈1操作
        PtrS -> Data[++(PtrS -> Top1)] = item;
    } else {
        //对 堆栈2操作
        PtrS -> Data[--(PtrS -> Top2)] = item;
    }
}

(2)出栈

ElementType Pop(struct DStack *PtrS, int Tag){
    //Tag作为区分两个堆栈的标志，取值为1,和2
    if(Tag == 1){
        //对 堆栈1操作
        if(PtrS -> Top1 == -1){
            printf("堆栈1空");
            return NULL;
        } else {
            return PtrS -> Data[(PtrS -> Top1)--];
        }
    } else {
        //对 堆栈2操作
        if(PtrS -> Top2 == MaxSize){
            printf("堆栈2空");
            return NULL;
        } else {
            return PtrS -> Data[(PtrS -> Top2)++];
        }
    }
}

2.2.3 堆栈的链式存储实现

栈的链式存储结构 是 一个单向链表，叫做 链栈。
插入和删除操作 只能在 链栈 的栈顶进行。

typedef struct Node{
    ElementType Data;
    struct Node *Next;
}LinkStack;
LinkStack *Top;

(1)堆栈初始化（建立空栈）

LinkStack *createStack(){
    //构建 一个堆栈 的头结点，返回指针
    LinkStack *S;
    S = (LinkStack *)malloc(sizeof(struct Node));
    S -> Next = NULL;
    return S;
}

(2)判断 堆栈S 是否为空

int IsEmpty(LinkStack *S){
    //空 返回1，非空 返回0
    return (S -> Next == NULL);
}

(3)入栈

void Push(ElementType item, LinkStack *S){
    //将item 压入 堆栈S
    struct Node *TmpCell;
    TmpCell = maoolc(sizeof(struct Node));
    TmpCell -> Element = item;
    TmpCell -> Next = S -> Next;
    S -> Next = TmpCell;
}

(4)出栈

ElementType Pop(LinkStack *S){
    //删除栈顶元素 并返回S的栈顶元素
    struct Node *FirstCell;
    ElementType TopElem;
    if(IsEmpty(S)){
        printf("堆栈空");
        return NULL;
    } else {
        FirstCell = S -> Next;
        S -> Next = FirstCell -> Next;
        TopElem = FirstCell -> Element;
        free(FirstCell);
        return TopElem;
    }
}

2.2.4 堆栈应用：表达式求值

  • 回忆 堆栈 实现 后缀表达式 求值：
    从左到右 读入 后缀表达式 的各项（运算数 或 运算符）
    • 1.运算数：入栈
    • 2.运算符：从 堆栈中 弹出 适当数量 的运算数，计算并结果入栈
    • 3.最后，栈顶元素 就是表达式的结果值

中缀表达式求值

基本策略 ：将中缀表达式 转换为 后缀表达式，求值
[例子]: 2+9/3-5 后缀表达式为：293/+5-

  • 1.运算数相对顺序不变
  • 2.运算符号顺序改变
    • 需要存储“等待中”的运算符号
    • 将当前运算符号 与 “等待中” 的 后一个运算符号 比较

[例]a(b+c)/d 后缀表达式： abc+d/

中缀表达式 如何 转换为 后缀表达式
从头到尾读取 中缀表达式的每个对象，不同对象做不同处理：

  • 1.运算数：直接输出；
  • 2.左括号：压入堆栈
  • 3.右括号：将栈顶的运算符弹出并输出，直到遇到左括号（左括号直接出栈，不输出）
  • 4.运算符
    • 优先级 大于 栈顶运算符，则压入栈；
    • 优先级 小于等于 栈顶运算符，将 栈顶运算符 弹出并输出；再比较 新的栈顶运算符，直到该运算符 优先级 大于 栈顶运算符 为止，然后将 该运算符压入栈；
  • 5.若各对象处理完毕，把堆栈中 存留的 运算符 一并输出

堆栈的其他应用

  • 函数调用 及 递归 实现
  • 深度优先搜索
  • 回溯算法
  • ……

什么是线性表

  • 多项式表示问题的启示：

    • 1.同一个问题可以有不同的表示（存储）方式
    • 2.有一类共性问题：有序线性序列的组织和管理

  • 线性表：由同类型数据元素构成有序序列的线性结构

    • 长度：表中元素个数
    • 空表：线性表中没有元素
    • 表头：表起始位置；表尾：表结束位置

线性表的抽象数据类型描述

线性表的顺序存储实现

主要操作的实现

1.初始化（建立空的顺序表） 

List *MakeEmpty(){
    List *PtrL;
    PtrL = (List*)malloc(sizeof(List));
    PtrL->Last = -1;
    return PtrL;
}

2.查找 

int find(ElementType X, List *PtrL){
    int i = 0;
    while(i <= PtrL->Last && PtrL->Data[i]!=X){
        i++;
    }
    if(i > PtrL->Last){
        return -1;
    }else{
        return i;//找到后返回存储位置
    }
}

查找成功的平均次数为(n+1)/2，平均时间性能为O(n)

3.插入：第i( l<=i <= n+1)个位置上插入一个值为X的新元素
下表是i-1的位置赋值为X，下表i到SIZE-1的部分右移。
右移时应从右往左操作。

void Insert(ElementType X, int i, List *PtrL){
    int j;
    if(PtrL->Last == MAXSIZE - 1){
        printf("表满");
        return;
    }

    if(i < 1 || i > PtrL -> Last +2){
        printf("位置不合法");
        return;
    }

    for(j = PtrL->Last; j >= i - 1; j--){
        PtrL->Data[j + 1] = PtrL->Data[j];//对a[i]-a[n]倒序赋值
    }
    PtrL->Data[i -1] = X;//插入新元素
    PtrL->Last++;//Last指向最后元素
    return;
}

4.删除：删除表的第i( l <= i <= n)个位置上的元素
删除下表为i-1的元素，后边的元素全部前移一个位置。从左往右移

void Delete(int i, List *PtrL){
    int j;
    if(i < 1 || i > PtrL -> Last +1){
        printf("不存在第%d个元素"， i);
        return;
    }

    for(j = i; i <= PtrL -> Last; j++){
        PtrL -> Data[j - 1] = PtrL -> Data[j];//将a[i+1]到a[n]顺序向前移动
    }
    PtrL-> Last--;//Last指向最后一个元素
    return;
}

平均移动次数是(n-1)/2
平均时间性能为O(n)

不要求 逻辑上相邻 的两个元素 物理上也相邻：通过 “链” 建立起数据元素之间的逻辑关系。
数组是逻辑相邻，且物理相邻的。
节点表示：

主要操作的实现

1.求表长 

int Length(List *PtrL){
    List *p = PtrL;//p指向表的第一个节点
    int j = 0;
    while(p){
        p = p -> Next;
        j++; //当前p指向第j个节点

时间性能为O(n)

2.查找
(1)按序号查找：FindKth

(2)按值查找：Find

    List *p = PtrL;
    while(p != NULL && p -> Data != X){

平均时间性能为O(n)

** 3.插入（在低 i-1( 1<=i<=n+1)个结点后插入一个值为X的新结点）

  • 1.先构造一个新结点，用s指向；
  • 2.再找到链表的第i-1个结点，用p指向；
  • 3.然后修改指针，插入结点（p之后插入新结点是s）
List *Insert(ElementType X, int i, List *PtrL){
    List *p, *s;
    if(i == 1){
        //新结点插入在表头
        s = (List *)malloc(sizeof(List));//申请空间，填装结点
        s -> Data = X;
        s -> Next = PtrL;
        return s;//返回新表头指针
    }

    p = FindKth(i -1, PtrL);//查找第i-1个结点
    if(p == NULL){
        printf("第i-1个不存在，不能插入");
        return NULL;
    }else{
        s = (List *)malloc(sizeof(List));//申请空间，填装结点
        s -> Data = X;
        s -> Next = p -> Next;//新结点插入在第i-1个结点后边
        p -> Next = s;
        return PtrL;
    }
}

** 4.删除（删除链表的第i( 1<=i<=n)个位置上的结点）

  • 1.先找到链表的第i-1个结点，用p指向；
  • 2.再用指针s指向要被删除的结点（p的下一个结点）
  • 3.然后修改指针，删除s所指向的结点
  • 4.最后释放s所指结点的空间
List *Delete(int i, List *PtrL){
    List *p, *s;
    if(i == 1){
        //删除表的第一个结点
        s = PtrL;//s指向第1个结点
        if(PtrL != NULL){
            PtrL = PtrL -> Next;
        }else{
            return NULL;
        }
        free(s);
        return PtrL;
    }

    p = FindKth(i - 1, PtrL);//查找第i-1个结点
    if(p == NULL){
        printf("第%d个结点不存在", i - 1);
        return NULL;
    }else if(p -> Next == NULL){
        printf("第%d个结点不存在", i);
        return NULL;
    }else{
        s = p -> Next;//s指向第i个结点
        p -> Next = s -> Next;//从链表删除
        free(s);//释放被删除的结点
        return NULL;
    }
}

平均查找次数是n/2
平均时间性能是O(n)

typedef struct GNode{
    int Tag;//标志域：0表示结点是单元素，1表示结点是广义表
    union{
        //子表指针域Sublist与单元素数据域Data复用，即共用存储空间
        ElementType Data;
        struct GNode *SubList;
    }URegion;
    struct GNode *Next;//指向后继结点
}GList;

通过Tag来确认数据是Data还是SubList

多重链表
多重链表：链表中的节点可能同时隶属于多个链

  • 多重链表中结点的指针域会有多个，如前面例子包含了Next和SubList两个指针域
  • 但包含两个指针域的链表并不一定是多重链表，比如在双向链表不是多重链表

多重链表的用途
树、图 这样相对复杂的数据结构都可以采用多重链表的方式存储

多重链表表示稀疏矩阵

Head是行或列的头结点

Term是稀疏矩阵的非零项
左上角的Term结点，是入口结点，表示有4行5列，7个非零项

数据结构1 基本概念——1-3最大子列和问题

1.最大子列和问题

算法1 

int MaxSubseqSum1(int A[], int N){
    int ThisSum, MaxSum = 0;
    int i, j, k;
    for(i = 0; i < N; i++){
        for(j = i; j < N; j++){
            ThisSum = 0;
            for(k = i; k <=j; k++){
                ThisSum += A[k];
                if(ThisSum > MaxSum)
                    MaxSum = ThisSum;
            }
        }//j循环结束
    }//i循环结束
    return MaxSum;
}

复杂度 T(N) = O(N^3)

算法2 

int MaxSubseqSum2(int A[], int N){
    int ThisSum, MaxSum = 0;
    int i, j;
    for(i = 0; i < N; i++){
        ThisSum = 0;
        for(j = i; j < N; j++){
            ThisSum += A[j];//对于相同的i，不同的j，只要在j-1次循环的基础上累加1即可
            if(ThisSum > MaxSum){
                MaxSum = ThisSum;
            }
        }
    }
    return MaxSum;
}

复杂度 T(N) = O(N^2)

算法3：分而治之

算法4:在线处理 

int MaxSubseqSum4(int A[], int N){
    int ThisSum, MaxSum;
    int i;
    ThisSum = MaxSum = 0;
    for(i = 0; i < N; i++){
        ThisSum += A[i];//向右累加
        if(ThisSum > MaxSum){
            MaxSum = ThisSum;//发现更大的，更新当前结果
        }else if(ThisSum < 0){
            ThisSum = 0;//不能使后边的部分和增大，抛弃
            }
    }
    return MaxSum;
}

T(N) = O(N)
举例[-1,3，-2,4，-6,1,6，-1]
MaxSum = 7

“在线”的意思是指 每输入一个数据 就进行 及时处理，在任何一个地方中止输入，算法都能正确给出当前的解。

数据结构1 基本概念——1-2什么是算法

基本概念

1.2什么是算法？

1.算法

  • 一个有限指令集
  • 接受一些输入（有些情况下不需要输入）
  • 产生输出
  • 一定在有限步骤后终止
  • 每一条指令必须
    • 有充分明确的目标，不可以有歧义
    • 计算机能处理的范围之内
    • 描述应该抽象，不依赖于任何一种计算机语言以及具体实现手段

例1.选择排序算法的伪码描述

void SelectionSort(int List[], int N)
{
    /*将N个整数List[0]...List[N - 1] 进行从小到大排序*/
    for(i = 0; i < N; i++){
        MinPosition = ScanForMin(List, i, N - 1);
        /*从List[i]到List[N - 1]中找最小元，并将其位置赋给MinPosition*/
        Swap(List[i], List[MinPosition]);
        /*将未排序部分的最小元换到有序部分的最后位置*/
    }
}

抽象的体现：

- List是数组还是链表？
- Swap用函数还是用宏去实现？

2.什么是好的算法？

  • 空间复杂度S(n) —— 根据算法写成的程序在执行时 占用存储单元的长度。这个长度往往与输入数据的规模有关。空间复杂度过高的算法可能导致使用的内存超限，造成程序非正常中断。
  • 时间复杂度T(n) —— 根据算法写成的程序在执行时耗费时间的长度。这个长度往往也与输入数据的规模有关。时间负责度过高的低效算法可能导致我们在有生之年都等不到运行结果。

1.1 例2 递归打印出1-N

void PrintN(int N)
{
    if(N){
        PrintN(N - 1);
        printf("%d\n", N);
    }
    return;
}

S(N)= CN,空间复杂度是一个N的函数，C是一个常亮，空间复杂度随着N的增加而变大。
每个程序可用空间有限，当N很大，需要的空间大于当前程序拥有的空间时，程序异常退出。不能输出正确结果。

1.1 例3 多项式求值

double f(int n, double a[], double x)
{
    int i;
    double p = a[0];
    for(i = 1; i <= n; i++){
        p += (a(i) * pow(x, i));
    }
    return p;
}

执行了多少次乘法？
for循环执行了n次，
pow(x,i)是求x的i次方，执行了i-1次乘法，
每一次循环中，执行了i次乘法。
一共执行了(1+2+…+n) = (n ^2 + n) / 2 次乘法。
时间复杂度 T(n) = C1 n^2 + C2 n

double f(int n, double a[], double x)
{
    int i;
    double p = a[n];
    for(i = n; i > 0; i--){
        p = a[i - 1] + x * p;
    }
    return p;
}

执行了多少次乘法？
每次循环执行1次乘法，执行了n次循环，一共执行了n次乘法。
时间复杂度 T(n) = C n

什么是最好的算法？

  • 在分析一般算法的效率时，我们经常关注下边两种复杂度
    • 最坏情况复杂度 T worst (n)
    • 平均复杂度 T avg (n)

我们更关注T worst

2.例2：写一个函数printN，输入正整数N，顺序打印出1-N的全部正整数。

void printN(int N)

void printN(inr N)

3.例3.计算给定多项式在给定点x处的值

计算两种方式耗时：
clock() : 捕捉从程序开始运行到clock()被调用时所耗费的时间，单位是clock tick，“时钟打点”
CLK_TCK : 机器时钟每秒所走的时钟打点数

#include <stdio.h>
#include <time.h>
clock_t start, stop;
double duration;

int main()
{
    start = clock();
    MyFunction(); /*测试函数*/
    stop = clock();
    duration = ((double) (stop - start)) / CLK_TCK;

    return 0;
}

运行一次，duration可能为0.
解决方式：重复运行多次，再取平均值。

解决问题方法的效率，跟算法的巧妙程度有关

总结

  • 数据对象在计算机中的组织方式：
    • 逻辑结构
    • 物理存储结构
  • 数据对象必定与一系列加在其上的操作相关联
  • 完成这些操作所用的方法就是算法

4. 抽象数据类型

  • 数据类型
    • 数据对象集
    • 数据集合相关联的操作集
  • 抽象： 描述数据类型的方法不依赖于具体实现
    • 与存放数据的机器无关
    • 与数据存储的物理结构无关
    • 与实现操作的算法和编程语言均无关
      即 只描述数据对象集和相关操作集“是什么”，不涉及“如何做到”的问题

例4：“矩阵”的抽象数据类型定义

  • 类型名称：矩阵
  • 数据对象集：一个MxN的矩阵

由MxN个三元组构成，其中a是矩阵元素的值，i是元素所在的行号，j是元素所在的列号。（a是抽象的，不管a具体是什么，二维数组？一维数据？十字链表？）

  • 操作集：对于任意矩阵 A、B、C 属于 Matrix，以及整数 i、j、M、N
    • Matrix Create(int M, int N) 返回一个M x N的空矩阵
    • int GetMaxRow(Matrix A) 返回矩阵A的总行数
    • int GetMaxCol(Matrix A) 返回矩阵A的总列数
    • ElementType GetEntry(Matrix A, int i, int j) 返回矩阵A的第i行、第j列的元素
    • Matrix Add(Matrix A, Matrix B) 如果A和B的行、列数一致，则返回矩阵C = A + B，否则返回错误标志。（C = A + B是抽象的，先按行加？先按列加？什么语言？）
    • Matrix Multiply(Matrix A, Matrix B) 如果A的列数等于B的行数，则返回矩阵C = A B，否则返回错误标志。
